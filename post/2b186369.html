1. AOP 概念

面向切面编程，就是将一些跟具体业务逻辑不相干的，但在每个业务模块中又必不可少的通用功能模块提取出来，统一做成通用的切面，在需要的地方切入即可

  • where: 何处切入（切入到哪个类的那个方法中）
  • when: 何时切入（方法调用前、后来执行切入代码）
  • what：切入为何（切入时具体做什么）

  • 切面类：Aspect 整个切入逻辑综合，包含切入点和切入处理
  • 切入点方法：PointCut 负责 where。
    • 切点分为execution方式和annotation方式。前者可以用路径表达式指定哪些类织入切面，后者可以指定被哪些注解修饰的代码织入切面。
    • 切入点一般为切面类中的一个方法，在方法上加上 @PointCut 注解，注解其中参数指定切入切面的位置表达式
  • 切入处理方法：Advice 负责 when 和 what，即何时切入，以及切入时要做的操作
    • Advice 一般是切面类中的一个方法，方法本身就是切入的处理方法，即切入激活时，要执行的切入操作，即我们所说的 What
    • 关于 when ，是由修饰 Advice 方法的注解决定的，注解名称决定执行切入的时机，注解的参数指定对应的切入点

前面代码编写好后，后面这些由 spring 框架来执行

  • 连接点：JointPoint 被切入程序的切入位置，要跟切入点向对应
  • 织入（切入）: Weaving 切入这个动作，通过动态代理，执行切入操作

2. 相关注解

  • @Aspect：声明该类为一个切面类；

切面类定义好后，就可以在其中定义切点方法。

  • @Pointcut：定义一个切点，后面跟随一个表达式，表达式可以定义为切某个注解，也可以切某个 package 下的方法；

五种 When ：

  • @Before: 在切点之前，织入相关代码；
  • @After: 在切点之后，织入相关代码;
  • @AfterReturning: 在切点返回内容后，织入相关代码，一般用于对返回值做些加工处理的场景；
  • @AfterThrowing: 用来处理当织入的代码抛出异常后的逻辑处理;
  • @Around: 环绕，可以在切入点前后织入代码，并且可以自由的控制何时执行切点；

3. 切点表达式

3.1 execution表达式：

以 execution(* * com.mutest.controller...(..))) 表达式为例：

  • 第一个 * 号的位置：表示返回值类型，* 表示所有类型。
  • 包名：表示需要拦截的包名，后面的两个句点表示当前包和当前包的所有子包，在本例中指 com.mutest.controller包、子包下所有类的方法。
  • 第二个 * 号的位置：表示类名，* 表示所有类。
  • (..)：这个星号表示方法名， 表示所有的方法，后面括弧里面表示方法的参数，两个句点表示任何参数。
